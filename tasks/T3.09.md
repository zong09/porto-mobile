## Contract slice (CONTRACTS §12)
```dart
@freezed abstract class SettingsState { String displayCurrency; String language; }  // 'THB'|'USD', 'th'|'en'
@riverpod class SettingsNotifier extends _$SettingsNotifier {   // provider → settingsProvider
  FutureOr<SettingsState> build();
  Future<void> setCurrency(String v);      // 'THB'|'USD'
  Future<void> setLanguage(String v);      // 'th'|'en'
  Future<Map<String,dynamic>> exportToJson();
  Future<void> importFromJson(Map<String,dynamic> d);
}
```

## What to build
### `SettingsScreen` — `ConsumerWidget` watching `settingsProvider`
Per design-settings "Settings": hero title "ตั้งค่า" + on-device banner. Three `DividedCard` sections:
- **ทั่วไป**: สกุลเงินหลัก row → trailing shows `state.displayCurrency` (`฿ THB`/`$ USD`); tapping
  toggles via `ref.read(settingsProvider.notifier).setCurrency(<other>)`. ธีม row (static "สว่าง").
  ภาษา row → shows `state.language`; tapping toggles `setLanguage(th↔en)`.
- **ความปลอดภัย**: two toggle rows (Face ID, ซ่อนยอดเงิน) — **local UI state only** (no contract
  field); render the toggle switch (design-components §4), default Face ID ON / hide OFF.
- **ข้อมูลของคุณ**: **สำรองข้อมูลลงไฟล์** row → `onTap` calls a `Future<void> Function()? onExport`
  (default: `ref.read(settingsProvider.notifier).exportToJson()`); **นำเข้าข้อมูล** row → `onImport`
  (default: `importFromJson({})` is unsafe — instead default to a no-op callback the test overrides);
  **ลบข้อมูลทั้งหมด** row (destructive red; `onTap` → confirm dialog, action may be a callback).
  To make export/import testable, expose them as injectable callbacks on the widget:
  `SettingsScreen({VoidCallback? onExport, VoidCallback? onImport})` — default `onExport` triggers the
  notifier export; test passes recording callbacks.
- Footer version line.

### `ChartSheet` (`chart_sheet.dart`)
```dart
class ChartSheet extends StatefulWidget {
  final String title;                 // e.g. "Bitcoin · BTC"
  final List<PricePoint> history;     // PricePoint{ int t; double p; }
  final List<String> ranges;          // e.g. ['7D','30D','90D','1Y']
  final ValueChanged<String>? onRangeChange;
  const ChartSheet({super.key, required this.title, required this.history,
      required this.ranges, this.onRangeChange});
}
```
Per design-settings "Chart sheet": big current price (last point) + change pill; **AreaChart**
chart-sheet variant fed `history.map((e) => e.p).toList()` (brand line + brand fill + end dot,
~160px tall); **range pills** (local active state; tap → `onRangeChange`). Empty history → muted
"ไม่มีข้อมูลราคาย้อนหลัง". Present via `showPortoSheet(context, title: title, builder: ...)` (caller side).

## Fake-state test pattern (`test/settings_screen_test.dart`)
```dart
class _FakeSettings extends SettingsNotifier {
  final SettingsState _s; String? lastCurrency;
  _FakeSettings(this._s);
  @override FutureOr<SettingsState> build() => _s;
  @override Future<void> setCurrency(String v) async { lastCurrency = v; }
}
Widget _app(SettingsState s, {VoidCallback? onExport, VoidCallback? onImport}) => ProviderScope(
  overrides: [settingsProvider.overrideWith(() => _FakeSettings(s))],
  child: MaterialApp(home: SettingsScreen(onExport: onExport, onImport: onImport)));
```
Tests:
1. `testWidgets('shows current currency + language')`: state `displayCurrency:'THB', language:'th'`
   → both reflected; no exception.
2. `testWidgets('tapping currency row toggles THB↔USD')`: tap the row → assert the fake's
   `setCurrency` was called with `'USD'` (expose it, e.g. read the notifier and check `lastCurrency`).
   [The plan's "toggle" criterion.]
3. `testWidgets('export/import rows invoke callbacks')`: pass recording `onExport`/`onImport`; tap
   each row; assert both fired. [The plan's "export/import picker" criterion.]
4. `testWidgets('ChartSheet renders AreaChart from history')`: pump
   `ChartSheet(title:'BTC', history:[PricePoint(1,10),PricePoint(2,12),...], ranges:['7D','30D'])`
   in MaterialApp+Scaffold; `find.byType(AreaChart)` one; tapping a range pill fires `onRangeChange`.
5. `testWidgets('ChartSheet empty history shows no-data note')`.
