## Contract slice (CONTRACTS §10.2)
```dart
@freezed class TxRow           { Transaction tx; Asset asset; }
@freezed class TxGroup         { String date; List<TxRow> rows; }   // groups pre-sorted date DESC
@freezed class TransactionsState { List<TxGroup> groups; }
// provider: transactionsProvider (async). Mutations:

```
`Transaction` row fields: `id, assetId, side ('buy'|'sell'|'deposit'|'withdraw'), quantity, price,
fee, date ('YYYY-MM-DD'), createdAt`. `Asset` fields incl. `symbol, name, currency`.

## What to build
### `TransactionsScreen` — `ConsumerWidget` watching `transactionsProvider`
Per design-transactions "Transactions (list)":
- Hero: title "รายการ"; a summary line (e.g. count or Σ buy/sell — compute simply from groups);
  **filter pills** `ทั้งหมด · ซื้อ · ขาย · ฝาก/ถอน`. The active filter is local `useState`/StatefulWidget
  state; filtering is applied to the rows shown (`all` / `side=='buy'` / `side=='sell'` /
  `side∈{deposit,withdraw}`). A group with no matching rows after filtering is hidden.
- Sheet body: for each `TxGroup` → a `font 12/700 #A89A86` date header (format the ISO date; "วันนี้"
  handling optional — a plain date is acceptable) + a `DividedCard` of rows. Each row = `ListRowTile`:
  leading 38×38 tinted square keyed by `tx.side` (buy `↓` gain-bg; sell `↑` loss-bg; deposit `＋`
  gold-bg; withdraw `−` rose-bg — colors per design-transactions); title `"${sideLabel} ${asset.symbol}"`;
  subtitle `"${qty} @ ${price} · ${asset.name}"`; trailing signed amount
  (`Formatters.money(qty*price+fee, currency: asset.currency)` with sign by side) + time-from-createdAt
  is optional. `onTap` → open the form in edit mode (may pop for now).

### `TransactionSheet` (`transaction_sheet.dart`)
```dart
class TransactionSheet extends ConsumerStatefulWidget {
  final String side;             // 'buy'|'sell'|'deposit'|'withdraw'
  final List<Asset> assets;      // selectable assets (passed in; caller supplies)
  final Asset? initialAsset;
  final Transaction? existing;   // edit mode
  const TransactionSheet({super.key, required this.side, required this.assets,
      this.initialAsset, this.existing});
}
```
Per design-transactions "Transaction form": asset selector; จำนวน + ราคาต่อหน่วย inputs; **มูลค่ารวม**
banner = `qty*price + fee` (live-computed); date field; fee field; note (optional/ignored if no
model field). Save → `ref.read(transactionsProvider.notifier).addTransaction(assetId, side, quantity,
price, fee, date)` (or `saveTransaction` in edit mode). Inputs are entered in **display currency**;
if you convert to native, use `CurrencyConverter.convert` — but since price is stored in the asset's
native currency and this task has no live FX, keep entry == native and note it. Validate qty>0, price≥0.

## Fake-state test pattern (`test/transactions_screen_test.dart`)
```dart
class _FakeTx extends TransactionsNotifier {
  final TransactionsState _s; _FakeTx(this._s);
  @override FutureOr<TransactionsState> build() => _s;
}
```
Helpers build `Transaction`, `Asset`, `TxRow`, `TxGroup` (plain ctors).
Tests:
1. `testWidgets('rows grouped under date headers in order')`: state with 2 groups (dates D1, D2) →
   expect both date labels found and the first group's row appears before the second's (verify order
   via `tester.getTopLeft`), no exception.
2. `testWidgets('filter pill ซื้อ shows only buys')`: state mixing buy+sell → tap "ซื้อ" → a sell
   row's title is gone, a buy row's title remains.
3. `testWidgets('form computes มูลค่ารวม = qty*price+fee')`: pump `TransactionSheet(side:'buy',
   assets:[a], initialAsset:a)`; enter qty=2, price=100, fee=5 → expect a widget showing 205 (the
   total) is found (format-tolerant: find text containing '205').
4. `testWidgets('save calls addTransaction')`: use a fake notifier that records the call; tap save;
   assert the recorded assetId/side/quantity match.
