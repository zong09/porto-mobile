## Goal
A hand-painted area/line chart `CustomPainter` (no chart library) for the net-worth hero sparkline
and price-history charts. Mirrors the web app's smooth SVG area path.

## API to build
```dart
/// A smooth area/line chart. Values are plotted evenly across the width,
/// oldest→newest, with the line optionally filled down to the baseline.
class AreaChart extends StatelessWidget {
  final List<double> values;
  final Color line;
  final Color? fill;        // null = stroke only (row sparkline)
  final bool showEndDot;    // draws a filled dot at the last point
  final double strokeWidth; // default 2.5
  const AreaChart(this.values, {super.key, required this.line, this.fill,
      this.showEndDot = false, this.strokeWidth = 2.5});
  @override
  Widget build(BuildContext context) =>
      CustomPaint(painter: AreaChartPainter(values, line: line, fill: fill,
          showEndDot: showEndDot, strokeWidth: strokeWidth), child: const SizedBox.expand());
}

class AreaChartPainter extends CustomPainter { ... }
```

## Algorithm (language-neutral)
Let `n = values.length`, `w = size.width`, `h = size.height`.
1. If `n == 0` → paint nothing, return. If `n == 1` → treat as a flat line at mid-height.
2. `minV = min(values)`, `maxV = max(values)`, `range = maxV - minV`.
   Top inset `topPad = h * 0.08`, baseline `= h` (bottom).
3. For each i: `x = w * i / (n - 1)`; if `range == 0` → `y = h * 0.5`, else
   `y = topPad + (1 - (values[i] - minV) / range) * (h - topPad)`.
4. Build a **smooth** path through the points using Catmull-Rom→cubic conversion
   (for each segment use control points
   `c1 = p[i] + (p[i+1]-p[i-1])/6`, `c2 = p[i+1] - (p[i+2]-p[i])/6`, clamping indices at the ends),
   `path.cubicTo(c1.dx,c1.dy, c2.dx,c2.dy, p[i+1].dx,p[i+1].dy)`. Start with `path.moveTo(p[0])`.
5. **Fill** (only if `fill != null`): clone the line path, `lineTo(w, h)`, `lineTo(0, h)`, `close()`;
   paint with `Paint()..style = PaintingStyle.fill..color = fill!`.
6. **Stroke** the line path: `Paint()..style = stroke..strokeWidth = strokeWidth..strokeCap = round
   ..color = line`.
7. **End dot** (if `showEndDot`): `canvas.drawCircle(p.last, 4, Paint()..color = line)`.
8. `shouldRepaint`: return true if `values`, `line`, `fill`, `showEndDot`, or `strokeWidth` differ
   (compare lists with `listEquals`).

## Variants (documented for callers; the painter just takes colors)
- Hero on gradient: `line: Colors.white, fill: Colors.white.withValues(alpha: 0.14), showEndDot: true`.
- Row sparkline: `line: AppColors.gain` (or `AppColors.loss`), `fill: null, strokeWidth: 2`.
- Chart sheet on cream: `line: AppColors.brand, fill: <brand @ ~0.18>`.

> Note: use `Color.withValues(alpha: x)` (not the deprecated `withOpacity`) to keep `flutter analyze`
> clean on the pinned SDK.

## Test vectors (`test/area_chart_test.dart`)
Use `tester.pumpWidget` wrapping the chart in a sized box, then assert it paints without throwing.
1. `testWidgets('paints an 8-point hero series')`: pump
   `SizedBox(width: 358, height: 74, child: AreaChart(const [10,12,11,15,14,18,17,22], line: Colors.white, fill: Color(0x24FFFFFF), showEndDot: true))`
   inside a `MaterialApp`/`Center`; `expect(tester.takeException(), isNull)` and
   `expect(find.byType(CustomPaint), findsWidgets)`.
2. `testWidgets('stroke-only sparkline, no fill')`: pump with `fill: null`, `[1,2,3]`; no exception.
3. `test('shouldRepaint true when values change')`:
   `expect(AreaChartPainter(const [1,2], line: Colors.white).shouldRepaint(AreaChartPainter(const [1,3], line: Colors.white)), isTrue);`
4. `testWidgets('empty list paints nothing without error')`: pump `AreaChart(const [], line: Colors.white)`; no exception.
