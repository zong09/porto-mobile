## Contract slice (CONTRACTS §10.1)
```dart
@freezed class AssetNode      { Asset asset; PositionSummary position; }   // position: quantity, avgCost, totalCost, realizedPnl, direction
@freezed class PortfolioNode  { Portfolio portfolio; List<AssetNode> assets; } // Portfolio: id, name, color(0-5), sortOrder
@freezed class PortfoliosState{ List<PortfolioNode> nodes; }
// provider: portfoliosProvider (async). Mutations on the notifier:
//   addPortfolio({name,color}), renamePortfolio(id,name), recolorPortfolio(id,color),
//   deletePortfolio(id), addAsset({portfolioId,type,symbol,name,currency,cgId?,yahooSymbol?,manualPrice?,direction}),
//   saveAsset(Asset), deleteAsset(id)
```
`Asset` row fields: `id, portfolioId, type, symbol, name, currency ('THB'|'USD'), cgId?, yahooSymbol?,
manualPrice?, direction, sortOrder`. Use `AppColors.palette[portfolio.color]` for the card accent.

## What to build
### `PortfoliosScreen` (list) — `ConsumerWidget` watching `portfoliosProvider`
Per design-portfolios "Portfolios (list)":
- Hero: title "พอร์ตของฉัน" + "＋ สร้างพอร์ต" (opens create sheet → `addPortfolio`); total value =
  sum over nodes of each portfolio value (value of a node = Σ `assetNode.position.totalCost` is a
  rough stand-in — **use `position.quantity * position.avgCost`**? No: there is no live price in this
  state → display **cost basis** `Σ position.totalCost` and label it faithfully, OR if the design
  needs market value, note it's unavailable here). Keep it simple: show `Σ totalCost` as the total.
- Allocation bar (design-components §5) with one segment per node (width = node cost / total),
  colors `AppColors.palette[node.portfolio.color]`, + legend.
- One `PlainCard`/`ListRowTile` per node: leading 42×42 tinted square with a `DonutChart`
  (slices = each asset's `position.totalCost`); title `portfolio.name`; subtitle
  `${assets.length} สินทรัพย์`; trailing value = node cost. `onTap` → push `PortfolioDetail(node)`.
- Dashed "＋ สร้างพอร์ตใหม่" card at the end.

### `PortfolioDetailScreen` (same file) — takes a `PortfolioNode`
Per design-portfolios "Portfolio detail": back button; portfolio name; value; allocation bar over its
assets; a featured asset card (first asset) with a row **AreaChart** sparkline (feed a small synthetic
or `position`-derived series is fine — no history in state; use `const [] `→ skip chart if empty) and
**ซื้อเพิ่ม / ขาย** buttons (open the transaction form — for now `onTap` callbacks that may be no-ops
/ pop); remaining assets in a `DividedCard`; realized-P/L banner summing `position.realizedPnl`.

### `AssetSheet` (`asset_sheet.dart`)
```dart
class AssetSheet extends ConsumerStatefulWidget {
  final String portfolioId;      // for create
  final Asset? existing;         // non-null = edit mode (currency LOCKED)
  const AssetSheet({super.key, required this.portfolioId, this.existing});
}
```
Body per design-portfolios "Asset sheet": AssetType segmented chips; symbol + name inputs; currency
segmented (**disabled when `existing != null`**); optional manualPrice/cgId/yahooSymbol. Save button
calls `ref.read(portfoliosProvider.notifier).addAsset(...)` (create) or `saveAsset(existing.copyWith(...))`
(edit). Inline-validate: name non-empty, symbol matches `^[A-Za-z0-9.\-^=]{1,15}$`.

## Fake-state test pattern (`test/portfolios_screen_test.dart`)
```dart
class _FakePortfolios extends PortfoliosNotifier {
  final PortfoliosState _s; _FakePortfolios(this._s);
  @override FutureOr<PortfoliosState> build() => _s;
}
```
Build helpers for `Portfolio`, `Asset`, `PositionSummary`, `AssetNode`, `PortfolioNode` (plain ctors).
Tests:
1. `testWidgets('list renders one card per portfolio node')`: 2 nodes → expect both names found,
   `find.byType(ListRowTile)` count ≥ 2, no exception.
2. `testWidgets('tapping a card opens detail')`: tap first card → detail shows portfolio name +
   an asset symbol.
3. `testWidgets('AssetSheet create mode allows currency choice')`: pump `AssetSheet(portfolioId:'p1')`
   inside a ProviderScope+MaterialApp+Scaffold; the currency control is enabled (tappable).
4. `testWidgets('AssetSheet edit mode locks currency')`: pump with `existing:` an Asset (currency
   'USD'); assert the currency control is disabled / cannot change (e.g. wrapped in `IgnorePointer`
   or `AbsorbPointer`, or a disabled style flag you can find). This is the plan's
   "currency-locked-after-create" criterion.
